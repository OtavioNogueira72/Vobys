# Teste Técnico – Analista de Parametrização

Este repositório contém a **solução proposta para o teste técnico** da vaga de **Analista de Folha de Pagamento e RH**, com foco em **parametrização de sistemas**, **regras de negócio** e **lógica de lançamento de rubricas**, de forma independente de fornecedor ou ERP específico.

---

## Objetivo da Solução

Demonstrar:
- Capacidade de **compreender regras de negócio** de folha de pagamento;
- Organização lógica para **parametrização de rubricas**;
- Separação clara entre **regra de lançamento** e **regra de cálculo**;
- Atenção à **consistência de dados**, **validação** e **governança do processo**;
- Comunicação clara entre áreas técnicas e funcionais.

---

## Escopo Abordado

A solução contempla:
- Agrupamento dos diferentes **regimes de vínculo** (RPPS e RGPS);
- Definição das **rubricas de desconto** necessárias (IRRF e Previdência);
- Regras lógicas para determinar **quando cada rubrica deve ser lançada**;
- Premissas de vínculo válido por competência;
- Exemplo de **pseudocódigo** para ilustrar a lógica de decisão.

> Importante: o foco é a **lógica de parametrização**, não o cálculo financeiro das rubricas.

## Abordagem Metodológica

A lógica adotada segue os seguintes princípios:
1. **Validação do vínculo** antes de qualquer processamento;
2. Inicialização de uma lista vazia de rubricas;
3. Inclusão de rubricas somente quando as **condições são atendidas**;
4. Separação entre **dados cadastrais**, **regras legais** e **decisão sistêmica**;
5. Preocupação com **homologação e prevenção de retrabalho**.

---

## Observações Finais

- A solução foi construída com base em **boas práticas de parametrização**, aplicáveis a ERPs de mercado (SAP, TOTVS, Senior, etc.);
- Não utiliza dados reais ou sensíveis;
- O modelo apresentado é facilmente adaptável a diferentes sistemas e contextos organizacionais.
